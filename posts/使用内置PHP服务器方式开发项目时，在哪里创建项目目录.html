使用内置PHP服务器方式开发项目时，在哪里创建项目目录

Posted Mar 28th, 2026 by author
Stuffren
in the category "技术教程"

刚才我通过手动方式成功将php8.4.19 NTS版本安装到了E:\php 问题：当我选择通过“启动PHP内置服务器的方式”来开发本地项目时，请举例说明，我应该选择在哪里建立项目目录呢？

kimi

项目目录选择原则

PHP 内置服务器会把你指定的目录作为网站根目录（Document Root），所以：

项目目录 ≠ PHP 安装目录

推荐在独立位置创建项目文件夹，保持结构清晰。

推荐目录结构示例

方案一：集中式（推荐初学者）

E:\
├── php\                    ← PHP 程序本身（已安装）
│   ├── php.exe
│   └── ...
│
└── www\                    ← 所有网站项目放这里
    ├── project1\           ← 项目1
    │   ├── index.php
    │   └── ...
    │
    ├── project2\           ← 项目2
    │   ├── index.php
    │   └── ...
    │
    └── test-app\           ← 你的新应用
        ├── index.php
        ├── api\
        └── assets\

启动命令：

# 进入项目目录
cd E:\www\test-app

# 启动服务器（当前目录作为根目录）
php -S localhost:8000

# 或者明确指定根目录
php -S localhost:8000 -t E:\www\test-app

方案二：分散式（按项目类型）

E:\
├── php\                    ← PHP 程序
├── projects\               ← 工作项目
│   ├── company-website\
│   └── internal-tool\
├── experiments\            ← 实验/学习
│   └── php-learning\
└── temp\                   ← 临时测试
    └── quick-test\

实战：创建第一个项目

# 1. 创建项目目录
mkdir E:\www\my-first-app -Force
cd E:\www\my-first-app

# 2. 创建入口文件
@'
<?php
phpinfo();
'@ | Out-File -FilePath "index.php" -Encoding utf8

# 3. 启动服务器
php -S localhost:8000

然后浏览器访问：http://localhost:8000

关键要点总结

问题 答案
PHP 安装在哪 E:\php（程序文件，不碰它）
项目放哪里 另外创建，如 E:\www\项目名
如何启动 在项目目录运行 php -S localhost:8000
能否同时运行多个项目 可以，用不同端口：php -S localhost:8001

核心原则：PHP 程序 和 你的代码 分开存放，互不干扰。
